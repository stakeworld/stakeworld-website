When running a validator node it is important to have an action plan in place when there are problems with your node. For example you node server goes down, or a database gets corrupted.

:::info
stakeworld doesn't copy session keys and keeps backup nodes available so that if a node fails we can quickly bind the stash to a new node with minimal downtime and no risk of slashing. 
:::
